Deploy a controller on a Linux VM in a public cloud. These steps cover Azure and AWS.

## 1.1 Create a VM to be used as the controller

  defaultValue="Azure"
  values={[

>
<TabItem value="Azure">

It is easier to use a **resource group** to organize and manage all your resources for this demo.

1. Create a **resource group** and change into that resource group.
2. Use the **+ Create** button to create a resource.
3. Azure will take you to the **Marketplace** screen. In the search bar, type in **Ubuntu Server**.
4. Choose "**Ubuntu Server 22.04 LTS**".

    ![Diagram](/img/public_cloud/Create-Azure1.jpg)

5. On the **Create a virtual machine** screen:
    - The **Subscription** and **Resource group** should already be filled.
    - In the **Instance details** section, enter the **VM name**.
    - Select the **Region** to host your VM.
    - Leave default **Availability options** and **Security type**(Standard).
    - Leave the selected image **Ubuntu Server 22.04 LTS x64 Gen2**.
    - For the Size, choose the appropriate size for your application. For this demo, **Standard_B2s(2CPU,4 GB)** size was used.

    ![Diagram](/img/public_cloud/Create-Azure2.jpg)

6. Configure authentication:
    - Choose **Authentication type** to log in to the VM. **Logging in with a password is insecure**.
    - Enter a username (**remember the username, you will need it to log in to the VM**), or leave the default user **azureuser**.
    - Choose your SSH key.
    - For **inbound ports**, select the ssh. You can add an extra port based on your OpenZiti setup.
    - You can leave everything default.
7. Press **Review + create**.

    ![Diagram](/img/public_cloud/Create-Azure4.jpg)

8. After the Validation passed, press **Create** to create VM.

    ![Diagram](/img/public_cloud/Create-Azure5.jpg)

9. **Important:** If you are creating the network controller, you need to associate a DNS name to the public IP of your VM. You can do this from "Virtual machine" page.

    ![Diagram](/img/public_cloud/Create-Azure6.jpg)

</TabItem>
<TabItem value="AWS">

1. Sign in to the **AWS console**.
2. Go to the **EC2 dashboard**.
3. Click on the **Instances**.
4. **Launch the instances**.

    ![Diagram](/img/public_cloud/Create-AWS1.jpg)

5. Fill the **Name** of your instance.

    ![Diagram](/img/public_cloud/Create-AWS2.jpg)

6. Select the image and instance type:
    - On the Quick start select **Ubuntu**.
    - Select the **Ubuntu Server 22.04 LTS**.
    - Leave the Architecture as **64-bit (x86)**.
    - Select the instance type **t2.medium**.

    ![Diagram](/img/public_cloud/Create-AWS3.jpg)

7. Configure the key pair and network settings:
    - In the **Key pair (login.)** section, choose the key name which you already created.
    - In the **Network settings** section, choose the VPC for your VM.
    - Select the subnet.
    - Select **Enable** for **Auto assign public IP**.

    ![Diagram](/img/public_cloud/Create-AWS4.jpg)

8. Create a security group:
    - Click on **Create security group** to allow traffic into your VM.
    - Name the security group.
    - Create the Firewall rule based on the ports you configured for the controller.
    - Allow the same TCP port you configured (e.g., 1280) for the controller along with SSH.

    **Security Group For the controller.**
    ![Diagram](/img/public_cloud/Firewall-AWS-Controller.jpg)

    **Security Group for the Router**
    ![Diagram](/img/public_cloud/Firewall-AWS-ER.jpg)

9. Under **Configure storage** section, choose at least **20** GiB storage space.
10. Now click on **Launch instance**.

</TabItem>
<TabItem value="GCP">

1. Sign in to the GCP console.
2. Go to **COMPUTE ENGINE** dashboard.
3. Click on **CREATE INSTANCE**.

    ![Diagram](/img/public_cloud/Create-GCP1.jpg)

4. Configure the VM as follows:
    - **Name** of the VM
    - **Region** and **Zone**
    - Choose **e2-medium** for **Machine type**

    ![Diagram](/img/public_cloud/Create-GCP4.jpg)

5. Change the OS image:
    - Hit **CHANGE** at the "Boot disk" section to change the OS image.
    - On the "Boot disk" page, choose **PUBLIC IMAGES**.
    - Choose **Ubuntu** as the Operating system.
    - Select **Ubuntu 22.04 LTS, x86** Version.
    - Hit **SELECT** to complete the selection.

    ![Diagram](/img/public_cloud/Create-GCP5.jpg)

6. Configure networking:
    - Open **Advanced Options**, and then open **Networking**.
    - **Highly recommended:** assign a Network tags. This will help you to configure the firewall later.
    - **For local GW VM**, **Enable** IP forwarding at this time. You will not be able to change this setting from the console after the VM is created.

    ![Diagram](/img/public_cloud/Create-GCP6.jpg)

7. **Optional** you can reserve static external IP ADDRESS under **Networking/Network interfaces** section. Reserving static IP is useful for the Network controller in case you have to shut down the VM.

    ![Diagram](/img/public_cloud/Create-GCP7.jpg)

8. Now click on **CREATE** to create the virtual machine.

</TabItem>
<TabItem value="DigitalOcean">

1. Sign in to the Digital Ocean console and create a **Droplets** from the dropdown menu on the upper right-hand side.

    ![Diagram](/img/public_cloud/Create1.jpg)

2. On the "Create Droplets" screen, choose "**Ubuntu**", version "**22.04**".
3. For the Size, choose the appropriate size for your application. For this guide, a smaller size was used.

    ![Diagram](/img/public_cloud/Create2.jpg)

4. Choose an ssh-key to log in to the VM. (We highly discourage login to the VM using Password.)
5. Then **Create Droplet**.

    ![Diagram](/img/public_cloud/Create3.jpg)

</TabItem>
<TabItem value="OCI">

1. Sign in to the **ORACLE Cloud** console.
2. Go to Home > Dashboard.
3. Click on **Instances** (Under **Compute** category).
4. Create an instance.
5. Configure the instance:
    - Name the instance.
    - Choose the compartment.
    - Select the Availability domain under Placement.
    - Leave the security disabled.

    ![Diagram](/img/public_cloud/Create-OCI1.jpg)

6. Select the image and shape:
    - Under the "Image and shape selection", click **Change image** icon.
    - Select the **Ubuntu** icon.
    - Select the **Canonical Ubuntu 22.04**.
    - Select any image build. And press **Select image**.
    - Under the "Shape" selection, choose **Change shape**.
    - Change to **2** OCUPs and **4** GB memory.

    ![Diagram](/img/public_cloud/Create-OCI2.jpg)

7. Configure networking:
    - Select your **Primary network** and **Subnet**.
    - For **Public IPv4 address**, check **Assign a public IPv4 address**.

    ![Diagram](/img/public_cloud/Create-OCI3.jpg)

8. Configure SSH keys and boot volume:
    - In the **Add SSH keys** section, choose how you want your ssh keys generated. **Logging in with a password is insecure**.
    - Leave default boot volume.
9. Now click on **Create**.

    ![Diagram](/img/public_cloud/Create-OCI4.jpg)

</TabItem>
<TabItem value="IBM">

1. Sign in to the IBM cloud.
2. Go to dashboard.
3. Click on **Create resource +** on the top right.
4. Filter on "Compute" on the Category (on the left).
5. Choose **Virtual Server for Classic**.

    ![Diagram](/img/public_cloud/Create-IBM1.jpg)

6. Configure the virtual server:
    - Leave the "Type of virtual server" as **Public**.
    - Give it your **Hostname**.

    ![Diagram](/img/public_cloud/Create-IBM2.jpg)

7. Select the location, profile, and image:
    - Select the location.
    - Select the profile, the size of **B1.2x4** is adequate for our exercise.
    - Choose the SSH key.
    - On the "Operating system" select the **Ubuntu**. Select the version **22.04**.

    ![Diagram](/img/public_cloud/Create-IBM3.jpg)

8. Leave everything else default except **Add-ons**, turn the **Firewall** features on.

    ![Diagram](/img/public_cloud/Create-IBM4.jpg)

9. Alternatively, you can select **allow_all** under the "Public security group", and leave the Add-on Firewall feature off. **This is not recommended as it opens your VM for attacks**.
10. Press **Create** on the right side menu to create the VM.

Azure's firewall is blocking all incoming access to the VM. You will need to open the ports you configured for Ziti services. Here is an example of the firewall ports.

![Diagram](/img/public_cloud/Controller-Firewall-Azure.jpg)

</TabItem>
<TabItem value="AWS">

You must open the same ports in the AWS Security Group that you configured Ziti to listen for. Here is an example.

![Diagram](/img/public_cloud/Firewall-AWS-Controller.jpg)

</TabItem>

<TabItem value="GCP">

GCP’s default firewall is blocking all incoming access to the VM. You will need to open the same ports you specified for the controller and ZAC (if you plan to use ZAC). Here is an example of the firewall ports if you used the default ports.

1. Go to your VM screen, click on the **Network interfaces** name (i.e. nic0).
2. Click on **Firewall** menu on the left side to bring up the firewall screen.
3. On the firewall screen, click on **+ CREATE FIREWALL RULE** to create new rules.
4. Configure the firewall rule:
    - Give a meaningful **name** to your firewall rule.
    - Choose your **Network**.
    - Use traffic direction **Ingress**.
    - Action **Allow**.
    - Targets, you can use "All instances in the network" (if you did not specify "Network tags" for your VM). In this example, we are using **Specified target tags** option.
    - Enter **Target tags** for your VM. In this example, our tag is **nc**.
    - Enter Source IPv4 ranges: **0.0.0.0/0**.
    - For the controller, you must allow the same **TCP** ports you configured Ziti to listen for along with the SSH port (**22**).
5. Hit **CREATE** to create rules.

    ![Diagram](/img/public_cloud/Controller-Firewall-GCP.jpg)

6. The firewall rule also shows up on your "Network interface details" screen. From your VM screen, click on the **Network interfaces** name (i.e. nic0).

    ![Diagram](/img/public_cloud/Controller-Firewall-GCP2.jpg)

</TabItem>
<TabItem value="DigitalOcean">

DigitalOcean by default does not set up the firewall for the VM.
</TabItem>
<TabItem value="OCI">

Oracle cloud by default blocks all incoming traffic to the VM. You will need to open the ports you specified for the controller and ZAC (if you plan to use ZAC). 

First, create a security group:

1. From the **Networking** category, select the **Virtual cloud networks**.
2. Select the VCN your VM is in.
3. On the left side menu, select the **Network Security Group**.
4. Select **Create Network Security Group**.
5. Name the security group and select the next.
6. Now create rules for ingress traffic:
    - Port 22/TCP for SSH
    - OpenZiti ports, e.g., 1280/tcp
    - Also create a rule to allow all traffic outbound (**Egress**).

    Following is the example of the Security Group for the controller:

    ![Diagram](/img/public_cloud/Firewall-OCI1.jpg)

7. After the security group is created, attach it to the instance.
8. From the "Instance details" screen, select **Edit** under the **Network security groups** section.
9. Select the security group from the dropdown and press **Save changes**.

    ![Diagram](/img/public_cloud/Firewall-OCI2.jpg)

:::note
Oracle Cloud also uses Security Lists (on the subnet) to marshal the traffic. Make sure the setting under Security Lists is not conflicting with your security group rules.
:::

:::note
After the security group configuration, `ufw` may not work correctly on the VM. Turn on `ufw` and restart the VM after configuring the security group.
:::

1. Turn on `ufw` so traffic can reach the VM.
2. Set up `ufw` to allow traffic for OpenZiti ports:

```text
sudo ufw enable
sudo ufw allow 1280/tcp
sudo shutdown -r 0
```


</TabItem>
<TabItem value="IBM">

If you turn on the firewall feature, you will need to configure firewall rules.

1. Open the Instance detail screen.
2. Find the **Firewall details** at the bottom right. Open it.
3. Add the following rules:
    - ssh: port 22/TCP
    - OpenZiti ports, e.g., 1280/tcp
    - Deny rules to deny all other traffic
4. Make sure the firewall is active. It should display **Processing all rules** if it is active.

    ![Diagram](/img/public_cloud/Firewall-IBM-controller.jpg)

</TabItem>
</Tabs>

## 1.3 Log in and set up the controller

<TabItem value="Azure">

1. Once the VM is created, we can get the IP address (and the DNS name) of the VM from the Virtual machine screen.
2. Log in to the VM by using defined user "username" (default username is azureuser) and the private sshkey:

```text
ssh -i <private_key> <username>@<ip>
or
ssh -i <private_key> <username>@<dns-name>
```

Use a fully qualified domain name (FQDN) to [deploy the controller](/how-to-guides/deployments/10-linux/10-controller/10-deploy.mdx).

</TabItem>
<TabItem value="AWS">

1. Once the VM is created, we can get the IP address (and the DNS name) of the VM from the Instance detail screen.
2. **NOTE:** DNS name is only available if you enabled **DNS Hostnames** under VPC.
3. Log in to the VM with username "ubuntu":

```text
ssh -i <private_key> ubuntu@<ip>
or
ssh -i <private_key> ubuntu@<dns-name>
```
</TabItem>
<TabItem value="GCP">

Once the VM is created, we can log in through **SSH** button on the VM instances screen.

![Diagram](/img/public_cloud/GCP-login1.jpg)

</TabItem>
<TabItem value="DigitalOcean">

1. Once the VM is created, we can get the IP address of the droplet from the Resources screen.
2. Log in to the VM with username "root" and IP address:

```text
ssh root@<ip>
```

</TabItem>
<TabItem value="OCI">

1. Once the VM is created, we can get the IP address of the VM from the instance details screen.
2. Log in to the VM with username "ubuntu" and the IP address:


</TabItem>
