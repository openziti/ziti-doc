# Get started with OpenZiti

Start an OpenZiti overlay in just a couple of minutes and start exploring. The `ziti edge quickstart` command runs a 
short-lived controller and router together in a single process, perfect for learning OpenZiti and exploring zero trust 
firsthand. When you cancel the process (using `ctrl+c`), the overlay cleans up after itself. When you're ready to go 
further and want to make a long-lived overlay, check out the [deployment guides](@openzitidocs/category/deployments) 
to stand up a network that will last.

## 2. Run the quickstart

By default, the controller and router are configured to use the machine's hostname as the address advertised to clients.
This often works fine for local dev or on a local network, but it might not be sufficient. Use the `--ctrl-address` and 
`--router-address` flags to set an advertised address that every device you want on the overlay can actually reach.

```bash
ziti edge quickstart \
  --ctrl-address "quickstart.demo.openziti.io" \
  --router-address "quickstart.demo.openziti.io"
```

That's it. The controller and router will both start up and be ready to secure your traffic.

## 3. Verify using `ziti`

After about 10 seconds, your overlay will be up and running. Log in to confirm the controller's management API is ready for you to
start exploring using the `ziti` command. Authenticate to the controller to explore the API using the following command.

```bash
ziti edge login --username admin --password admin
```

After logging in, verify that the `router-quickstart` edge router is online.

```bash
$ ziti edge list edge-routers
```

<Details>
<summary>Optional flags</summary>

| Flag                | Default      | Purpose                                                          |
|---------------------|--------------|------------------------------------------------------------------|
| `--ctrl-address`    | *(hostname)* | advertised address for the control plane and API                 |
| `--ctrl-port`       | `1280`       | listen port for the control plane and API                        |
| `--router-address`  | *(hostname)* | advertised address for the integrated router                     |
| `--router-port`     | `3022`       | listen port for the integrated router                            |
| `--username` / `-u` | `admin`      | admin username                                                   |
| `--password` / `-p` | `admin`      | admin password                                                   |
| `--home`            | *(temp dir)* | write config to a permanent directory so state survives restarts |
| `--no-router`       | —            | start the controller only, without an integrated router          |
| `--verbose`         | —            | show additional output                                           |

</Details>
